1. Overview

TikTok's open platform is a set of REST APIs. Almost everything is built around OAuth 2.0: a TikTok user must log in to your app and grant permission (scopes) before you can read their data. You then call the APIs with that user's access_token.

Key rule: To pull a user's data (profile, followers, video engagement), that user must log in to your system and authorize the requested scopes. You cannot pull a random influencer's private stats without their login. The only no-login option is the Research API, which is restricted to approved non-profit / academic research.

2. Authentication Flow (Login Kit)

  1. Authorize — Redirect the user to TikTok's authorization page. They log in & tap Allow.
  2. Get code — TikTok redirects back to your redirect_uri with a code.
  3. Exchange token — Your server swaps the code for an access_token + refresh_token.
  4. Call APIs — Use Authorization: Bearer {access_token} to call Display API.
  5. Refresh — When the token expires, use the refresh_token to get a new one.

3. OAuth APIs

3.1 Authorization Request REDIRECT

Meaning: Sends the user to TikTok to log in and approve your app. This is the "Login with TikTok" button. Returns a one-time code to your redirect URI.
GET https://www.tiktok.com/v2/auth/authorize/
# Build this URL and open it in the user's browser
https://www.tiktok.com/v2/auth/authorize/
  ?client_key=CLIENT_KEY
  &scope=user.info.basic,user.info.profile,user.info.stats,video.list
  &response_type=code
  &redirect_uri=https://yourapp.com/callback
  &state=random_csrf_string

3.2 Exchange Code for Access Token POST

Meaning: Your backend trades the one-time code for an access_token (used to call APIs) and a refresh_token (used to renew it).
POST https://open.tiktokapis.com/v2/oauth/token/
curl --location --request POST 'https://open.tiktokapis.com/v2/oauth/token/' \
  --header 'Content-Type: application/x-www-form-urlencoded' \
  --data-urlencode 'client_key=CLIENT_KEY' \
  --data-urlencode 'client_secret=CLIENT_SECRET' \
  --data-urlencode 'code=AUTH_CODE_FROM_REDIRECT' \
  --data-urlencode 'grant_type=authorization_code' \
  --data-urlencode 'redirect_uri=https://yourapp.com/callback'

Returns: access_token, refresh_token, open_id, expires_in, refresh_expires_in, scope, token_type.

3.3 Refresh Access Token POST

Meaning: Access tokens expire (~24h). Use the refresh token to get a new access token without making the user log in again.
POST https://open.tiktokapis.com/v2/oauth/token/
curl --location --request POST 'https://open.tiktokapis.com/v2/oauth/token/' \
  --header 'Content-Type: application/x-www-form-urlencoded' \
  --data-urlencode 'client_key=CLIENT_KEY' \
  --data-urlencode 'client_secret=CLIENT_SECRET' \
  --data-urlencode 'grant_type=refresh_token' \
  --data-urlencode 'refresh_token=REFRESH_TOKEN'

3.4 Revoke Access POST

Meaning: Disconnects the user / invalidates the token (e.g. when a user unlinks their TikTok account from your app).
POST https://open.tiktokapis.com/v2/oauth/revoke/
curl --location --request POST 'https://open.tiktokapis.com/v2/oauth/revoke/' \
  --header 'Content-Type: application/x-www-form-urlencoded' \
  --data-urlencode 'client_key=CLIENT_KEY' \
  --data-urlencode 'client_secret=CLIENT_SECRET' \
  --data-urlencode 'token=ACCESS_TOKEN'

4. Display API (for your logged-in users)

Works only for users who logged in to your app and granted the scope. This is the main API for an influencer app.

4.1 Get User Info GET

Meaning: Returns the logged-in user's profile — name, avatar, bio, and (with stats scopes) follower / following / likes / video counts.
GET https://open.tiktokapis.com/v2/user/info/?fields=...
curl --location 'https://open.tiktokapis.com/v2/user/info/?fields=open_id,union_id,avatar_url,display_name,bio_description,profile_deep_link,is_verified,follower_count,following_count,likes_count,video_count' \
  --header 'Authorization: Bearer ACCESS_TOKEN'
FieldMeaningScope required
open_id / union_idUnique user IDs in your appuser.info.basic
display_namePublic nicknameuser.info.basic
avatar_urlProfile picture URLuser.info.basic
bio_descriptionProfile bio textuser.info.profile

is_verifiedBlue-check verified statususer.info.profile
follower_count👥 Number of followersuser.info.stats
following_countAccounts they followuser.info.stats

4.2 List User's Videos POST

Meaning: Returns a paginated list of the logged-in user's own videos, including engagement metrics (views, likes, comments, shares) per video.
POST https://open.tiktokapis.com/v2/video/list/?fields=...
curl --location --request POST 'https://open.tiktokapis.com/v2/video/list/?fields=id,title,video_description,duration,cover_image_url,share_url,embed_link,create_time,view_count,like_count,comment_count,share_count' \
  --header 'Authorization: Bearer ACCESS_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{ "max_count": 20 }'
# Use the returned "cursor" + "has_more" to page through all videos

4.3 Query Specific Videos POST

Meaning: Look up engagement for specific video IDs (up to 20). Useful to refresh stats of known videos. Only returns videos owned by the authorized user.
POST https://open.tiktokapis.com/v2/video/query/?fields=...
curl --location --request POST 'https://open.tiktokapis.com/v2/video/query/?fields=id,title,view_count,like_count,comment_count,share_count,cover_image_url,share_url' \
  --header 'Authorization: Bearer ACCESS_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{ "filters": { "video_ids": ["123456789", "987654321"] } }'
Video fieldMeaning

Scope for 4.2 & 4.3: video.list. Limit: up to 20 video IDs per request.

5. Research API (any public user — research only)

Access restricted: Requires a separate TikTok approval and is limited to non-profit academic research in the US/EU. Commercial influencer apps are usually rejected. Listed here for completeness.

5.1 Query User Info POST

Meaning: Look up a public user's stats by username — without them logging in.
POST https://open.tiktokapis.com/v2/research/user/info/?fields=...
curl --location --request POST 'https://open.tiktokapis.com/v2/research/user/info/?fields=display_name,bio_description,avatar_url,is_verified,follower_count,following_count,likes_count,video_count' \
  --header 'Authorization: Bearer ACCESS_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{ "username": "some_creator" }'

5.2 Query Videos POST

Meaning: Search public videos by conditions (hashtag, keyword, region, date) and read their engagement (views, likes, comments, shares).
POST https://open.tiktokapis.com/v2/research/video/query/?fields=...
curl --location --request POST 'https://open.tiktokapis.com/v2/research/video/query/?fields=id,video_description,create_time,region_code,view_count,like_count,comment_count,share_count,username,hashtag_names' \
  --header 'Authorization: Bearer ACCESS_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{ "query": { "and": [ { "operation": "EQ", "field_name": "username", "field_values": ["some_creator"] } ] }, "start_date": "20260101", "end_date": "20260131", "max_count": 20 }'

6. What is NOT available

❌ No Live Stream API. TikTok's official platform has no API to start a livestream, or to read live chat / gifts / viewer counts. There are no live webhook events either. The only ways people read live data are unofficial libraries (TikTok-Live-Connector, TikTokLive) that reverse-engineer the internal WebCast protocol — these are against TikTok's Terms of Service and can break at any time.
  • ❌ Go live / stream control — none
  • ❌ Live chat, gifts, real-time viewers — none (officially)
  • ❌ Reading another user's private data without their login — none
  • ❌ User-level total view count — not exposed (only per-video view_count)
